  <Tab title="Python">
    ## Build a Python Package

    This guide shows you how to build a Python package using the `pixi-build-python` backend.

    <Steps>
      <Step title="Create a Python project">
        Create a new directory with a basic Python package structure:

        ```bash
        mkdir my-python-package
        cd my-python-package
        ```

        Create a `pyproject.toml` file:

        ```toml pyproject.toml
        [project]
        name = "my-package"
        version = "0.1.0"
        description = "My awesome Python package"
        requires-python = ">=3.8"
        dependencies = [
            "requests>=2.28",
        ]

        [build-system]
        requires = ["hatchling"]
        build-backend = "hatchling.build"
        ```

        Create a simple Python module:

        ```bash
        mkdir -p my_package
        echo 'def hello(): return "Hello from my package!"' > my_package/__init__.py
        ```
      </Step>

      <Step title="Initialize Pixi project">
        Create a `pixi.toml` file to enable Pixi build:

        ```toml pixi.toml
        [workspace]
        preview = ["pixi-build"]
        channels = ["https://prefix.dev/conda-forge"]
        platforms = ["linux-64", "osx-64", "osx-arm64", "win-64"]

        [package]
        name = "my-package"
        version = "0.1.0"

        [package.build]
        backend = { name = "pixi-build-python", version = "*" }
        channels = ["https://prefix.dev/conda-forge"]
        ```

        <Info>
          The `preview = ["pixi-build"]` line enables the experimental pixi-build feature.
        </Info>
      </Step>

      <Step title="Build the package">
        Build your conda package:

        ```bash
        pixi build
        ```

        You'll see output like:

        ```
        ✓ Build completed successfully
        📦 Package: output/linux-64/my-package-0.1.0-py311_0.conda
        ```
      </Step>

      <Step title="Install and test">
        Install the package locally:

        ```bash
        pixi install
        ```

        Test it works:

        ```bash
        pixi run python -c "from my_package import hello; print(hello())"
        ```
      </Step>
    </Steps>

  <Tab title="Rust">
    ## Build a Rust Package

    This guide shows you how to build a Rust package using the `pixi-build-rust` backend.

    <Steps>
      <Step title="Create a Rust project">
        Create a new Rust library:

        ```bash
        cargo new --lib my-rust-package
        cd my-rust-package
        ```

        Your `Cargo.toml` should look like:

        ```toml Cargo.toml
        [package]
        name = "my-rust-package"
        version = "0.1.0"
        edition = "2021"

        [dependencies]
        serde = { version = "1.0", features = ["derive"] }
        ```
      </Step>

      <Step title="Add Pixi configuration">
        Create a `pixi.toml` file:

        ```toml pixi.toml
        [workspace]
        preview = ["pixi-build"]
        channels = ["https://prefix.dev/conda-forge"]
        platforms = ["linux-64", "osx-64", "osx-arm64", "win-64"]

        [package]
        name = "my-rust-package"
        version = "0.1.0"

        [package.build]
        backend = { name = "pixi-build-rust", version = "*" }
        channels = ["https://prefix.dev/conda-forge"]
        ```
      </Step>

      <Step title="Build the package">
        Build your conda package:

        ```bash
        pixi build
        ```

        The backend will automatically:
        - Extract metadata from `Cargo.toml`
        - Detect Rust compiler requirements
        - Build with cargo
        - Package the library files
      </Step>

      <Step title="Install the package">
        ```bash
        pixi install
        ```

        Your Rust library is now available as a conda package!
      </Step>
    </Steps>

    - Explore sccache optimization in the [Rust backend docs](/backends/pixi-build-rust)

  <Tab title="C/C++ (CMake)">
    ## Build a CMake Package

    This guide shows you how to build a C/C++ project using the `pixi-build-cmake` backend.

    <Steps>
      <Step title="Create a CMake project">
        Create a simple CMake project:

        ```bash
        mkdir my-cmake-package
        cd my-cmake-package
        ```

        Create `CMakeLists.txt`:

        ```cmake CMakeLists.txt
        cmake_minimum_required(VERSION 3.15)
        project(my-cmake-package VERSION 0.1.0)

        add_library(mylib src/mylib.cpp)
        target_include_directories(mylib PUBLIC include)

        install(TARGETS mylib DESTINATION lib)
        install(DIRECTORY include/ DESTINATION include)
        ```

        Create source files:

        ```bash
        mkdir -p src include
        echo '#include <string>\nstd::string hello() { return "Hello from CMake!"; }' > src/mylib.cpp
        echo '#include <string>\nstd::string hello();' > include/mylib.h
        ```
      </Step>

      <Step title="Add Pixi configuration">
        Create a `pixi.toml` file:

        ```toml pixi.toml
        [workspace]
        preview = ["pixi-build"]
        channels = ["https://prefix.dev/conda-forge"]
        platforms = ["linux-64", "osx-64", "osx-arm64", "win-64"]

        [package]
        name = "my-cmake-package"
        version = "0.1.0"

        [package.build]
        backend = { name = "pixi-build-cmake", version = "*" }
        channels = ["https://prefix.dev/conda-forge"]

        [package.build.config]
        compilers = ["cxx"]
        ```

        <Info>
          The `compilers = ["cxx"]` line tells the backend to include a C++ compiler.
        </Info>
      </Step>

      <Step title="Build the package">
        ```bash
        pixi build
        ```

        The backend will:
        - Configure with CMake
        - Build with Ninja
        - Install to the package prefix
        - Create the conda package
      </Step>

      <Step title="Install and use">
        ```bash
        pixi install
        ```

        Your C++ library is now available as a conda package!
      </Step>
    </Steps>

    ### Next Steps for CMake

    - Pass custom CMake flags in the [CMake backend docs](/backends/pixi-build-cmake)
    - Configure [cross-platform compilers](/concepts/compilers)

## Common Build Configurations

### Target-Specific Builds

Build for specific platforms:

```toml pixi.toml
[package.build.target.win-64.config]
compilers = ["c", "cxx"]

[package.build.target.linux-64.config]
env = { CFLAGS = "-O3" }
```

### Adding Dependencies

Specify build, host, and runtime dependencies:

```toml pixi.toml
[package.build-dependencies]
cmake = ">=3.15"
ninja = "*"

[package.host-dependencies]
libcurl = ">=7.0"

[package.run-dependencies]
libcurl = ">=7.0"
```

### Environment Variables

Set environment variables during build:

```toml pixi.toml
[package.build.config]
env = { 
    CUSTOM_FLAG = "enabled",
    BUILD_TYPE = "release"
}
```

## Troubleshooting

<Accordion title="Build fails with 'backend not found'">
  Make sure you have the `preview = ["pixi-build"]` line in your `pixi.toml` under `[workspace]`. The pixi-build feature is currently experimental.
</Accordion>

<Accordion title="Package version doesn't match">
  Ensure the `version` field in `[package]` matches the version in your language-specific manifest (`pyproject.toml`, `Cargo.toml`, etc.).
</Accordion>

<Accordion title="Dependencies not resolving">
  Check that:
  - Your `channels` list includes `https://prefix.dev/conda-forge`
  - Dependency names are conda package names, not PyPI/cargo names
  - You're using the correct dependency type (build/host/run)
</Accordion>

<Accordion title="Build succeeds but install fails">
  Run `pixi build` with verbose output:
  
  ```bash
  RUST_LOG=debug pixi build
  ```
  
  Check the generated recipe in `.pixi/build/work/<package-name>--<hash>/debug/recipe.yaml`
</Accordion>
